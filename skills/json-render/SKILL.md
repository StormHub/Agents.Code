# json-render

A complete framework for defining, generating, and rendering JSON specs into
UIs and videos. Perfect for AI-powered application builders that generate
interfaces from prompts.

## Package Overview

| Package | Purpose |
|---------|---------|
| [@json-render/core](./core/SKILL.md) | Schema definitions, catalogs, specs, and AI prompt generation |
| [@json-render/react](./react/SKILL.md) | React renderer for converting specs into React components |
| [@json-render/next](./next/SKILL.md) | Next.js renderer with full-stack support (routes, SSR, metadata) |
| [@json-render/shadcn](./shadcn/SKILL.md) | Pre-built shadcn/ui component library (36+ components) |

## When to Use json-render

- **Building AI-powered UI generators** that produce JSON specs from prompts
- **Creating component-driven applications** with AI-generated interfaces
- **Rapid prototyping** multi-page applications from structured JSON
- **Standardizing component catalogs** across React and Next.js projects
- **Generating video specs** or interactive UIs from JSON schemas

## Quick Workflow

1. **Define a Schema** (`@json-render/core`): Structure what your specs contain
2. **Create a Catalog** (`@json-render/core`): Map component names to definitions
3. **Generate Specs** (via AI): Use schemas to guide AI prompt generation
4. **Render** (`@json-render/react` or `@json-render/next`): Convert specs to UIs
5. **Use Components** (`@json-render/shadcn`): Leverage pre-built UI components

## Key Concepts

- **Schema**: TypeScript-defined structure for specs and catalogs
- **Catalog**: Maps component/action names to their definitions and validators
- **Spec**: JSON conforming to schema, generated by AI or manually
- **SpecStream**: JSONL format for streaming spec generation
- **Renderer**: Converts specs into React components or Next.js apps
